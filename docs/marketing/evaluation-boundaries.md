# Evaluation Boundaries

## Purpose

Make sandbox limits visible before a prospect treats evaluation output like production reliance.

## Required message

Evaluation is for integration testing, evidence review, and operating-model validation. It should not be used as the basis for regulatory compliance decisions.

## Boundaries

- Reference or fixture data may be used instead of production provider feeds.
- Evaluation output is not production reliance.
- Evaluation data has limited retention.
- Evaluation credentials do not grant production API keys, provider data, or tenant policy administration.

## Production readiness adds

Production onboarding requires data-feed configuration, tenant isolation, backup and restore posture, retention policy, operator review design, and customer-approved reliance language.
